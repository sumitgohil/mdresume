Good resume keywords are not a separate list you paste at the bottom. They should appear where they explain actual work:
skills, project scope, architecture decisions, tools used, and outcomes delivered.

Start with the job description. Pull out repeated technologies, seniority signals, and domain nouns. Then compare them
against your resume. If a keyword represents work you have done, add it to the most relevant bullet, project, or skills
line. If it does not represent real experience, leave it out.

For software roles, strong keyword coverage usually includes languages, frameworks, cloud services, databases, testing
tools, collaboration practices, and product domain terms. The goal is not volume. The goal is clear evidence that your
experience matches the hiring team's vocabulary.

Markdown makes this easier because you can keep a base resume and tailor a copy for each role. Diff the changes before
exporting so you can see whether the tailored version is genuinely stronger or just noisier.
