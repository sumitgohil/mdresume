A Markdown resume workflow works best when the content is structured first and styled second. Start with the header,
summary, skills, experience, projects, and education. Then tighten each bullet until it describes scope, action, and
result.

Frontmatter is useful for stable contact information, while the body remains focused on the story. This makes the same
resume source reusable across templates without copying content into multiple layouts.

The result is a resume that behaves more like source code: readable in plain text, easy to diff, and simple to publish.
